0002 — v1 quota dashboard (built)

2026-06-23 · Shipped · Umar + Claude

area: api + web credential-free issues: #1-#10, #14 closed

Problem

The plan, decisions, and the proven data pipeline existed; v1 needed to actually be a running dashboard. Two constraints shaped it: it must be credential-free (the owner did not want the app, or the coding agents, touching the Keychain), and it must look like one product with the docs.

Solution

A working vertical slice, built from scratch on the agreed stack:

  • api/ (FastAPI + uv). Pure, tolerant collectors: codex.py parses remaining windows from the rollout rate_limits; claude.py sums consumed tokens from the transcript message.usage. Read-only endpoints /api/quota, /api/events (SSE), /api/health, bound to 127.0.0.1, CORS to the web dev origin only. 100% test coverage on the pure logic (15 tests incl. degraded cases); ruff + mypy strict clean.
  • web/ (Next.js + TS). A branded dashboard: Codex 5h/weekly gauges (remaining % + reset countdown, ok/warn/crit colors, headroom nudge, throttle warning) and a Claude consumed-tokens meter, live via SSE with a fetch fallback and clean degraded / api-down states. DESIGN.md tokens, system fonts, no Tailwind.
  • Credential-free by construction. Both agents are read from local files only — no Keychain, no OAuth, no network to the vendors. The Claude remaining-window path is deferred. The verify-gate Stop hook (typecheck + mypy) is active and green.
Verified live on the owner's real data: Codex 5h 98% / weekly 57% remaining; Claude ~220M tokens this session (cache_read dominant). Screenshot shared with the owner.

Context

Run it: uv --directory api run uvicorn missioncontrol.main:app --host 127.0.0.1 --port 8787 and pnpm -C web dev, then open http://localhost:3000.

Deferred from v1 (issues left open): SQLite event-store persistence (#5; v1 computes snapshots live), out-of-tab OS notifications (#11; in-UI warning/nudge only), token + subprocess hardening (#12; N/A while credential-free), CSRF/session-token (#13; read-only API).

Next: v1.5 (agent watch-feed + consumed-cost), v2 (collaboration surfaces, Slack first), then work-threads and the parked AWS watch. See PLAN.md §8.
