2026-06-23 · Shipped (v2+ handed off) · Umar + Claude

area: api + web PLAN.md §5, §7, §8 issues: #5, #11, #12 (partial), #13, #15

Problem

v1 (entry 0002) shipped the credential-free quota dashboard but left the spine and the next rung of the roadmap unbuilt. The owner asked to "finish everything, one after another," so this session closed the open v1 follow-ups and built all of v1.5 — the agent watch-feed, consumed-cost metering, and jump-to-agent (the first command-exec surface).

Solution

Built on the existing testability split — pure logic at 100% coverage, I/O seams isolated and omitted from the gate (and still exercised by integration tests). 95 tests, 100% on pure logic, ruff + mypy + eslint + tsc all clean.

  • Redaction (#12, relevant slice). redact.py: mask token-shaped strings in text; recursively scrub auth-bearing keys/values before persistence. The load-bearing primitive that lets an event store satisfy "tokens never leak."
  • Event-store spine (#5). events.py + alerts.py (pure) derive scrubbed quota.snapshot / collector.status / alert.* rows from a snapshot; store.py persists them in one local SQLite DB (WAL, user_version migration, 0600 perms, retention cap). GET /api/timeline reads it; the SSE tick records + prunes; startup seeds. Nullable work_thread_id is already in the schema for v3.
  • Localhost hardening (#13). security.py: Origin allowlist + Host check (DNS-rebinding defense) enforced by middleware; a per-session token (secrets, constant-time compare) exposed at /api/session. A missing Origin/Host (curl, SSE) is allowed — not the threat.
  • OS notifications (#11). notify.py (pure): edge-triggered dedup so a desktop alert fires on the transition into throttle/headroom, not every poll; thresholds mirror the dashboard. osnotify.py seam: macOS osascript / Linux notify-send, absolute path, fixed argv, no shell, degrades to a quiet no-op.
  • Watch-feed (#15). transcript.py (pure): tolerant Claude + Codex JSONL → normalized turns {role, text, tools, ts}; thinking excluded; every text run through redact_text. GET /api/transcript; a roomy reading view at /watch with an agent toggle.
  • Consumed-cost metering (#15). pricing.py (pure): a dated, updatable rate card (per-1M-token rates captured 2026-06-23 from Anthropic public pricing; cache read 0.1×, cache write 1.25× input). The Claude collector sums tokens per model and reports estimated_cost_usd — surfaced as an estimate, never a billed figure. Unpriced model ⇒ None (vendor-neutral).
  • Jump-to-agent (#15) — first command-exec surface. POST /api/jump builds a claude --resume / codex resume command. Hardened: requires the per-session token (X-MC-Session) on top of the Origin/Host guard; agentcmd.py (pure) uses fixed argv and validates the session id against flag-injection (no leading -) and shell metachars. Local launch is opt-in (MC_ENABLE_JUMP=1) so non-invasive stays the default; the command is always returned so the UI can show/copy it.
Verified live in this container against its own ~/.claude transcript: /api/quota ~51.9M tokens / ~$45.94 est.; /api/transcript renders turns with tool chips + redaction; /api/timeline seeded; DB file 0600; /api/jump 403 without token / bad Origin, 200 + command with a valid token.

Context

Run it (unchanged): uv --directory api run uvicorn missioncontrol.main:app --host 127.0.0.1 --port 8787 + pnpm -C web dev. New env: MC_DB_PATH (event-store path), MC_ENABLE_JUMP=1 (enable the local agent launch). New seams in the coverage-omit list: store.py, osnotify.py, launch.py (each integration-tested anyway).

Watch out: the PostToolUse format hook runs ruff check --fix, which strips an import the moment it is momentarily unused — add an import and its usage together (or write the whole file), or it vanishes between edits.

Next (handed off — needs the owner's live env): v2 (#16) collaboration surfaces (Slack first) + the unified Timeline (the Timeline UI is pure-local on the event store and buildable now; Slack needs a workspace token in the app's runtime) + a generalized guarded action endpoint (the /api/jump token+Origin pattern is the template); v3 (#17) work-threads (schema already carries work_thread_id); later AWS (#18). These were not built here because the ephemeral cloud container has no SaaS/AWS credentials, so they cannot be verified with evidence — the right boundary for a fresh agent.
